Introduction to Movie management system

This is the movie manage system for assignment 4. It contains two systems.
The first system is implemented in file and the second system is implemented in database(sqlite3).
The two systems are sepreate, adding and deleting operation for one system will not affect the other.
To switch the two system, choose one in "Storage method" and click "Choose" button.
By default, the database management system is used.
Each movie has 3 attributes, name, rating and year.
Valid input for year and rate should be integers greater than 0. If any invalid input is found, an error page is returned.
Click the back on the returned page will bring you back to the starter page.
The server side manages a file called database.txt that stores the movie informations on the file management system
and manages a database in the moviedb.sqlite file using sqlite3 on the database management system.
Add and delete operation will modify the file or the database.

For the filter function:
To filter the movie, you can input only name, rate or year.
You can also input any combination of them to get a better result.
If nothing is inputted in the filter part, all the movies will be returned.
The filter is case insensitive.
When doing filter by name, if a movie name contains chars in the user input string in order (not necessaryly continuous), the movie is considered valid.

For the add function:
All three attributes must be presented. The movie will be added to "database.txt" file managed by server for the file management system.
The movie will be added to the movie database for the database management system.

For the delete function:
To delete a movie from the system, you can input only name, rate or year to delete a series of movies.
You can also input any combination of them to delete a specific movie.

For the list funciton:
When the list button is clicked, all the movies currently stored in database.txt file or in the movie database will be returned.
